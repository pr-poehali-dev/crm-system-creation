# Инструкция по исправлению проблемы с кэшем

## Проблема
После обновления CRM данные не отображаются из-за кэша браузера.

## Быстрое решение

### Вариант 1: Кнопка в интерфейсе
1. Откройте CRM систему
2. Найдите кнопку "Очистить кэш" в правом верхнем углу (рядом с кнопкой "Выйти")
3. Нажмите на неё
4. Страница автоматически перезагрузится через 1 секунду

### Вариант 2: Жёсткое обновление браузера
- **Windows/Linux**: Нажмите `Ctrl + F5` или `Ctrl + Shift + R`
- **Mac**: Нажмите `Cmd + Shift + R`

### Вариант 3: Очистка через DevTools
1. Откройте DevTools (`F12`)
2. Перейдите на вкладку "Network" / "Сеть"
3. Поставьте галочку "Disable cache" / "Отключить кэш"
4. Обновите страницу (`F5`)

## Что было исправлено

1. **Добавлены мета-теги в index.html:**
   ```html
   <meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate"/>
   <meta http-equiv="Pragma" content="no-cache"/>
   <meta http-equiv="Expires" content="0"/>
   ```

2. **Создана централизованная система API (`src/lib/api.ts`):**
   - Автоматическая очистка кэша через timestamp
   - Заголовки `Cache-Control: no-cache`
   - Опция `cache: 'no-store'` для всех запросов

3. **Созданы хуки для работы с данными (`src/hooks/use-crm-data.ts`):**
   - `useCRMClients()` - работа с клиентами
   - `useCRMBookings()` - работа с бронями
   - `useCRMVehicles()` - работа с автопарком
   - `useCRMPartners()` - работа с партнёрами

4. **Добавлена кнопка очистки кэша** в интерфейс

5. **Исправлена дата** - теперь отображается актуальная дата (18 января 2026)

## Проверка

После очистки кэша должны отобразиться:
- ✅ 14 клиентов
- ✅ 44 брони
- ✅ 3 автомобиля
- ✅ 1 партнёр

Все данные хранятся в базе данных и доступны через API.
